# Execution Plans

## Правило

В `active/` лежат задачи, которые можно брать в работу. После завершения файл переносится в `completed/` без изменения идентификатора. Если задача доведена до состояния финального коммита, перенос из `active/` в `completed/` должен происходить до этого коммита и попадать в тот же commit.

## Формат

Каждый plan обязан содержать:
- контекст;
- границы задачи;
- список ожидаемых артефактов;
- риски и открытые вопросы;
- верификацию.

## Текущая очередь

См. файлы `MLB-001` ... `MLB-010` в `active/`.
