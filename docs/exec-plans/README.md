# Execution Plans

## Правило

В `active/` лежат задачи, которые можно брать в работу. После завершения файл переносится в `completed/` без изменения идентификатора.

## Формат

Каждый plan обязан содержать:
- контекст;
- границы задачи;
- список ожидаемых артефактов;
- риски и открытые вопросы;
- верификацию.

## Текущая очередь

См. файлы `MLB-001` ... `MLB-008` в `active/`.
