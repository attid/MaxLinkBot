# MLB-004: Start Command And Reconcile

## Контекст

`/start` в первой версии это главный orchestration entrypoint. Он должен уметь либо запустить авторизацию, либо полностью сверить состояние MAX и Telegram для уже привязанного пользователя.

## Результат задачи

- handler `/start`;
- `RefreshReconcileService`;
- создание недостающих topics;
- восстановление удаленных topics;
- backfill последних `N` сообщений в новый или восстановленный topic.

## Шаги

1. Разделить ветки поведения:
   ignored, auth required, reconcile existing binding.
2. Получать список личных MAX-чатов пользователя.
3. Сверять его со стабильным topic mapping в SQLite.
4. Пересоздавать отсутствующие topics и обновлять mapping.
5. Выполнять backfill последних `N` сообщений.

## Риски и открытые вопросы

- Нужно отдельно продумать idempotency повторного `/start`.
- Важно не создавать дубликаты topics при гонках или повторных попытках.

## Верификация

- `/start` игнорируется для чужих user_id;
- `/start` запускает авторизацию, если binding нет;
- `/start` восстанавливает удаленный topic и подтягивает историю;
- повторный `/start` не ломает существующие mappings.
