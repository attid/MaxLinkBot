# MLB-005: MAX To Telegram Delivery

## Контекст

После reconcile система должна уметь постоянно переносить новые сообщения из личных MAX-чатов пользователя в соответствующие Telegram topics.

## Результат задачи

- poller новых MAX-сообщений;
- маршрутизация в topic по `telegram_user_id` и `max_chat_id`;
- автоматическое создание topic для нового личного чата;
- запись message links и audit events;
- fallback-описания для неподдерживаемых вложений.

## Шаги

1. Реализовать чтение новых сообщений MAX по активным binding.
2. Для каждого сообщения искать topic mapping только в контексте владельца.
3. Если личный чат новый, создать topic и сделать backfill.
4. Отправлять текст и поддерживаемые медиа в Telegram.
5. Для неподдерживаемых вложений писать явное служебное описание.

## Риски и открытые вопросы

- Нужны гарантии, что один и тот же MAX message не будет доставлен дважды.
- Нужно заранее определить минимальный набор поддерживаемых типов вложений в v1.

## Верификация

- новое текстовое сообщение из MAX попадает в правильный topic правильного пользователя;
- новый личный чат создает новый topic автоматически;
- неподдерживаемое вложение не теряется молча;
- чужой пользователь не может получить сообщение из другого binding.
