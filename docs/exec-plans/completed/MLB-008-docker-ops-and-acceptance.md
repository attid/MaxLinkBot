# MLB-008: Docker, Ops And Acceptance

## Статус

Ретроспективно помечено как выполненное.
Docker-setup, documented runtime config, README/runbooks и базовый acceptance path уже присутствуют в репозитории.

## Контекст

После сборки функционала нужен эксплуатационный слой: контейнерный запуск, конфиг, базовые runbooks и приемочная проверка end-to-end.

## Результат задачи

- Dockerfile и docker-compose или эквивалентный локальный запуск;
- единый конфиг приложения;
- README с quick start;
- runbooks по типовым инцидентам;
- финальный приемочный checklist для v1.

## Шаги

1. Собрать контейнерный запуск приложения и SQLite volume/path.
2. Проверить передачу обязательных env-переменных.
3. Описать первый запуск и сценарий локальной разработки.
4. Подготовить runbooks по `reauth_required`, отсутствию topics и проблемам доставки.
5. Собрать acceptance checklist по всем обязательным функциям v1.

## Риски и открытые вопросы

- Нужно заранее определить, где хранится SQLite в контейнере и как переживаются рестарты.
- Возможно, потребуется отдельный health endpoint или простая liveness-стратегия.

## Верификация

- приложение поднимается в Docker с документированным набором env;
- README позволяет новому исполнителю воспроизвести окружение;
- есть чеклист, покрывающий все обязательные требования первой версии.
