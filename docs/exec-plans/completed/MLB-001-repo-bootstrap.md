# MLB-001: Repo Bootstrap And Contracts

## Статус

Ретроспективно помечено как выполненное.
Каркас репозитория, базовые документы, структура `src/`/`tests/`, Docker-запуск и исходные контрактные артефакты уже присутствуют в репозитории.

## Контекст

В репозитории пока нет кодовой базы. Перед реализацией нужно создать минимальный каркас проекта, чтобы все последующие задачи опирались на одинаковую структуру каталогов, правила конфигурации и зафиксированный контракт интеграции с MAX API.

## Результат задачи

- каркас каталогов `src/`, `tests/`, `.linters/`;
- `README.md` с быстрым стартом;
- документ конфигурации окружения;
- ADR по выбору базовой структуры и зависимостей;
- зафиксированный выбор MAX-библиотеки: `maxapi-python`;
- фиксированный документ с известными ограничениями MAX API.

## Шаги

1. Создать директории будущего Python-приложения и тестов.
2. Зафиксировать базовый способ запуска в Docker.
3. Описать env-конфиг: bot token, allowlist, SQLite path, MAX API settings, poll interval, backfill count.
4. Оформить ADR по Clean Architecture для этого проекта и отдельно зафиксировать выбор `maxapi-python` как стартовой библиотеки для интеграции с MAX.
5. Добавить документ со списком неизвестных мест интеграции, которые надо проверить до кода.

## Риски и открытые вопросы

- Реальный контракт MAX API пока не хранится в репозитории.
- Нужно заранее решить, чем будет выражен allowlist в env: CSV, JSON или повторяемые переменные.
- Стоит сразу определить стратегию миграций SQLite, чтобы не переделывать позже.

## Верификация

- структура директорий соответствует `docs/architecture.md`;
- есть один источник правды по env-переменным;
- есть ADR, который объясняет выбранный каркас и его ограничения.
