# MLB-006: Telegram To MAX Delivery

## Статус

Ретроспективно помечено как выполненное.
Доставка `Telegram -> MAX` для topic messages, сервисный ответ вне topic и аудит исходящих операций уже реализованы; дополнительные media-фиксы были сделаны позже.

## Контекст

Вторая половина шлюза это корректная отправка пользовательских сообщений из Telegram topics обратно в соответствующие MAX-чаты без утечек и без ошибочной отправки текста из main chat.

## Результат задачи

- обработчик сообщений внутри topics;
- определение `max_chat_id` по `telegram_user_id + telegram_topic_id`;
- отправка в MAX API;
- служебный ответ на обычный текст вне topic;
- журнал успешных и неуспешных доставок.

## Шаги

1. Отделить main chat от topic-сообщений.
2. Для topic-сообщения находить mapping только через владельца.
3. Передавать текст и поддерживаемые медиа в MAX.
4. На текст вне topic отвечать сервисным сообщением о правильном сценарии.
5. Записывать результаты доставки и ошибки.

## Риски и открытые вопросы

- Нужно понять, есть ли ограничения Telegram topics в личном чате бота на практике.
- Следует заранее решить, как обрабатывать reply/forward в первой версии: поддержать минимально или отклонять явно.

## Верификация

- сообщение из topic уходит в правильный MAX-чат;
- текст вне topic не уходит в MAX и получает сервисный ответ;
- пользователь не может использовать чужой topic mapping.
