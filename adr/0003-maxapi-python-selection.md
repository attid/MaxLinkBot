# 0003: maxapi-python For MAX WebSocket Integration

## Контекст

Репозиторий уже содержит рабочую интеграцию с MAX через пакет `maxapi-python`, который импортируется в коде как `pymax`.
Документация и часть execution plans при этом отставали и продолжали ссылаться на старый выбор `love-apples/maxapi`, что перестало соответствовать реальному состоянию проекта.

По правилам `AI_FIRST.md` такие изменения должны фиксироваться через ADR, а не тихо переписываться задним числом.

## Решение

Актуальной библиотекой интеграции с MAX для этого репозитория фиксируется `maxapi-python`.

Правила использования:
- прямое использование внешнего клиента допускается только в `src/infrastructure/max/`;
- `domain/` и `application/` работают только через собственные порты и адаптеры проекта;
- в документации библиотека называется `maxapi-python`, а имя `pymax` используется только как имя Python-модуля и runtime-import path;
- устаревший ADR `0001` помечается как superseded.

## Последствия

- документация, README и execution plans должны ссылаться на `maxapi-python`, а не на старую библиотеку;
- импорт `from pymax import MaxClient` остается корректным и не требует переименования;
- дальнейшие изменения интеграционного контракта должны обновлять `docs/maxapi-python-contract.md`;
- при смене клиентской библиотеки в будущем потребуется новый ADR.

## Статус

Accepted
